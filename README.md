# StockScope-Lite

ChatGPTタスクでTSVを作成し、Googleスプレッドシートへ手動貼り付けした内容を、Apps Script経由のRemote JSONとしてGitHub Pagesに表示する半自動運用のダッシュボードです。

## ファイル構成

- `index.html`: 画面の土台
- `style.css`: デザイン
- `app.js`: データ読み込み、タブ切り替え、画面描画
- `config.js`: local / remote データの切り替え設定
- `data.json`: ローカル表示用サンプルデータ
- `apps-script/Code.gs`: GoogleスプレッドシートをJSON API化するApps Script
- `README.md`: 説明

## 半自動運用

1. ChatGPTタスクでTSVを作成する
2. ユーザーがGoogleスプレッドシートへ手動貼り付けする
3. Apps Scriptがスプレッドシート内容をJSON化する
4. GitHub PagesがRemote JSONを読み込んで表示する

GitHubで `apps-script/Code.gs` を変更しても、Google Apps Script本体には自動反映されません。変更後はGoogle Apps Scriptエディタへ手動で貼り替え、必要に応じて再デプロイしてください。

## スプレッドシートの役割

- `日次表示データ`: 日次画面の表示用データです。株価、前日比、対応の必要性、現在の結論、ニュース、見るポイントなどを読み込みます。
- `週次レビュー表示データ`: 週次画面の表示用データです。今週の予想、実際の値動き、一致度、来週方針などを読み込みます。
- `月次レビュー表示データ`: 月次画面の表示用データです。今月の見立て、実際の値動き、月間サマリー、強弱材料、来月方針などを読み込みます。
- `日次ログ` / `週次予想ログ` / `週次結果ログ` / `月次見立てログ` / `月次結果ログ`: 記録用シートです。StockScopeの画面表示は表示用シートから読み込みます。
- `保有銘柄マスター`: 銘柄名、証券コード、基本方針、表示順、有効/無効などの基本情報です。

`保有情報_非公開` はJSON化しません。保有株数、平均取得単価、評価損益、証券会社、口座区分、個人メモは返却JSONに含めません。

## 表示データ

Apps Scriptは、以下の表示用シートを正としてJSONを作ります。旧シート構成への後方互換はありません。

### 日次表示データ

主に反映される項目:

- 株価、前日比、前日比率
- 対応の必要性、現在の結論、自信度
- 一言、今日の見立て、株価サマリー、主な理由
- ニュース傾向、ニュース1〜3
- 今後見るポイント、方針変更トリガー、最短見通し
- 更新日時

`有効/無効` が `有効` の行だけを表示します。同じ証券コードで複数行ある場合は、日付が最新の行を優先します。日付が同じ場合は下の行を優先します。

### 週次レビュー表示データ

主に反映される項目:

- 対応の必要性、現在の結論
- 今週の予想、想定レンジ下限、想定レンジ上限
- 実際の値動き、実際レンジ下限、実際レンジ上限
- 一致度、当たった点、外れた点、次回に活かす点
- 主な理由、見るポイント、方針変更トリガー、来週方針
- 更新日時

### 月次レビュー表示データ

主に反映される項目:

- 対応の必要性、現在の結論
- 今月の見立て、想定レンジ下限、想定レンジ上限
- 実際の値動き、実際レンジ下限、実際レンジ上限
- 一致度、当たった点、外れた点、次回に活かす点
- 月間サマリー、主な理由、強弱材料、見るポイント、方針変更トリガー、来月方針
- 更新日時

月次レビュー表示データが空、または主要項目が未入力の場合は、画面には `月次データはまだありません` の空状態だけを表示します。

リスト項目は、セル内改行、`｜`、`|` 区切りに対応しています。株価の桁区切りカンマで分割しないため、`,` や `、` は区切り文字として扱いません。

ヘッダー名の前後に半角・全角スペースが入っていても読み取れるようにしています。

株価、前日比、前日比率、想定レンジ、実際レンジは、Apps Script側で `getDisplayValues()` を使ってスプレッドシート上の表示値をそのままJSONへ返します。たとえば `9,695円`、`+49円`、`-209円`、`+0.66%`、`9,300円` は文字列として返します。空セルの場合だけ `null` または空欄扱いになります。

## 関連ニュース

`ニュース1_ソース`、`ニュース2_ソース`、`ニュース3_ソース` にURLが入っている場合、対応するニュース見出しをクリック可能なリンクとして表示します。

URLではない場合は、媒体名として表示します。長いURLでもカード表示が崩れないようにしています。

`contentReference`、`xcontentReference`、`sourceReference`、`turn`、`cite`、`index` などの内部参照文字列は、ニュース見出し、内容、影響、ソース、判断文などに入っていても画面には表示しません。

## 前日比率

前日比率は小数第2位までで表示します。

例:

- `+0.66%`
- `-2.11%`
- `+0.40%`

`0.0066` のような小数で入っている場合は、必要に応じて `0.66%` として表示します。`0.66%` のようにすでに `%` が付いている場合も表示が崩れないように処理します。

## 週次・月次レビュー表示

GitHub Pagesの週次レビュー欄は、Apps Scriptが返す `weeklyReviews` を元に表示します。月次レビュー欄は `monthlyReviews` を元に表示します。

`weeklyReviews` は `週次レビュー表示データ`、`monthlyReviews` は `月次レビュー表示データ` から読み込みます。GitHub Pages側ではログを直接結合しません。スプレッドシート側で完成済みの表示用データをそのまま読み込みます。

全体タブでは週次レビューを要約表示します。個別銘柄タブでは、その銘柄に対応する週次レビューだけを表示します。

表示対応:

- 今週の予想: `今週の予想`
- 週間結果: `実際の値動き`
- 実際のレンジ: `実際レンジ下限` 〜 `実際レンジ上限`
- 一致度: `一致度`
- 当たった点: `当たった点`
- 外れた点: `外れた点`
- 次回改善: `次回に活かす点`
- 次回方針: `来週方針` / `来月方針`

未入力の項目はできるだけ表示しません。`週次レビュー表示データ` が空、または主要項目が空の場合は、週次レビュー欄に `週次レビューはまだありません` と表示します。`月次レビュー表示データ` が空、または主要項目が空の場合は、月次欄に `月次データはまだありません` と表示します。

## remote JSON 切り替え

`config.js` で読み込むデータ元を切り替えます。

```js
window.STOCK_DASHBOARD_CONFIG = {
  USE_REMOTE_DATA: false,
  REMOTE_DATA_URL: ""
};
```

- `USE_REMOTE_DATA=false` の場合は `data.json` を使用します。
- `USE_REMOTE_DATA=true` かつ `REMOTE_DATA_URL` が設定されている場合は、remote JSON の取得を試みます。
- 通常fetchに失敗した場合はJSONP方式で再取得します。
- fetchとJSONPの両方に失敗した場合のみ `data.json` にフォールバックします。
- `REMOTE_DATA_URL` にはGoogle Apps ScriptのWeb App URLを設定します。

## デバッグ表示

データ取得元の表示は通常画面では非表示です。

URLに `?debug=1` を付けた場合だけ、画面上にデータ取得元を表示します。

例:

```text
https://isan127.github.io/stock-dashboard/?debug=1
```

表示される値の例:

- `Remote JSON`
- `Remote JSONP`
- `Local data.json`
- `Remote失敗 → Local fallback`

`?debug=1` の場合は、データ取得元を確認できます。

## セキュリティ方針

GitHub Pages、JSON、README、`config.js` には以下を含めないでください。

- 保有株数
- 平均取得単価
- 評価損益
- 証券会社
- 口座区分
- GoogleスプレッドシートURL
- APIキー
- 個人メモ

Apps ScriptのWebアプリURLを使う場合も、READMEには実URLを書かないでください。`config.js` にも秘密情報や個人情報を直書きしないでください。

## Apps Script更新時の注意

GitHub上の `apps-script/Code.gs` を修正しただけでは、Google Apps Script本体には反映されません。

Apps Script側の挙動を変えた場合は、必ず以下を行ってください。

1. GitHubの `apps-script/Code.gs` の内容をコピーする
2. Google Apps Scriptエディタの `Code.gs` に貼り替える
3. 保存する
4. 必要に応じて新しいバージョンとして再デプロイする
5. Apps Script Web App URLを直接開き、JSONが更新されているか確認する
6. GitHub Pagesを `?debug=1` 付きで開き、取得元と件数、先頭銘柄の値を確認する

更新後のApps Script Web App JSONには、`meta.appsScriptVersion` として `new-display-sheets-v1` が入ります。この値が出ていない場合は、古い `Code.gs` または古いWebアプリのデプロイを見ています。

`new-display-sheets-v1` では、以下のようにスプレッドシートの表示値が文字列のまま返ります。

```json
{
  "price": "9,695円",
  "change": "-209円",
  "forecast": {
    "range": {
      "low": "9,300円",
      "high": "10,300円"
    }
  }
}
```

## Apps Script設置手順

1. Googleスプレッドシートを開く
2. 拡張機能 → Apps Script
3. `apps-script/Code.gs` の内容を貼り付ける
4. プロジェクト設定 → Script Properties に `SPREADSHEET_ID` を追加する
5. `SPREADSHEET_ID` の値に対象スプレッドシートIDを設定する
6. デプロイ → 新しいデプロイ → ウェブアプリ
7. 実行ユーザーを `自分` にする
8. アクセスできるユーザーを必要に応じて設定する
9. 発行されたWeb App URLを `config.js` の `REMOTE_DATA_URL` に入れる
10. `config.js` の `USE_REMOTE_DATA` を `true` に変更する
11. ダッシュボードを開いて表示確認する

## ローカル確認

`fetch` で `data.json` を読むため、ローカルで `index.html` を直接開くと動かない場合があります。その場合は簡易サーバーで確認します。

```bash
python3 -m http.server 8000
```

ブラウザで `http://localhost:8000` を開き、全体タブと銘柄別タブが切り替わるか確認してください。
